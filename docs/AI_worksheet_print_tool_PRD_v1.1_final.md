# AI 练习卷打印助手 PRD v1.1（极简点数与定价最终版）

> 产品定位：一个像计算器一样打开即用的 AI 教育工具。  
> 核心能力：输入要求或上传资料，自动生成可打印的 PDF / Word 练习卷。  
> 核心策略：免费强体验；只对“生成”扣点；PDF 下载不扣点；免费版 PDF 有水印，付费版无水印 / Word / 答案解析 / 整卷仿真。  
> 首版目标：上线一个极简工具，验证用户是否愿意为“可打印练习卷”和高级交付权益付费。

## 0. v1.1 本次整合结论

本版本根据最新讨论，对原 PRD 的定价、点数、免费/付费权益、分享奖励、整卷仿真消耗和开发要求做最终收敛。

### 0.1 最终原则

1. **只为生成扣点**：用户生成练习卷、错题同类题、上传资料生成、整卷仿真时扣点。
2. **PDF 下载不扣点**：生成完成后即可下载 PDF；免费版 PDF 有水印，付费版 PDF 无水印。
3. **Word 不按点数扣费**：Word 下载是套餐权益，不单独扣点；仅 Pro / Teacher 支持。
4. **整卷仿真固定 10 点**：MVP 不按页数、OCR、解析拆分计费；最多上传 5 页。
5. **分享规则极简**：每日首次分享到朋友圈，分享人奖励 1 点；好友通过邀请链接首次购买成功，邀请人奖励 5 点。
6. **国内和海外分开定价**：国内用人民币套餐和点数包；海外用美元订阅和 credits pack。
7. **商业 MVP 必须有极简数据库**：不长期保存上传文件，但必须保存用户、点数、订单、分享奖励、邀请奖励和生成流水。

## 1. 项目背景

当前用户可以通过豆包、ChatGPT、DeepSeek 等通用 AI 生成题目，但存在几个明显问题：

1. 普通家长、老师、学生不知道怎么写提示词。
2. 生成结果通常只是聊天文本，不像一份正式练习卷。
3. Word / PDF 导出不方便。
4. 答案页、解析页、题号、空白答题区、排版需要人工整理。
5. 上传试卷后想得到一份类似的新试卷，通用 AI 的流程不稳定。
6. 用户真正需要的不是“AI 聊天”，而是一份能直接打印、发给孩子/学生使用的练习卷。

因此，本项目不做重型教育平台、不做题库系统、不做学习管理系统，而是做一个极简 AI 工具：

> 输入知识点、题目、学习资料，或者上传 PDF / Word / 图片，生成一份可打印的练习卷，并支持 PDF / Word 下载。

## 2. 产品定位

### 2.1 一句话定位

**AI 练习卷打印助手：输入知识点或上传资料，30 秒生成一份可打印 PDF / Word 练习卷。**

### 2.2 英文定位

**AI Printable Worksheet Generator**  
Turn any topic, notes, or exam paper into a printable worksheet. Download as PDF or Word in seconds.

### 2.3 本质定位

本产品不是教育平台，而是一个工具站产品。

类比：

| 类型 | 代表 | 本产品对应 |
|---|---|---|
| 计算器工具 | Calculator | 输入数字，得到计算结果 |
| AI 图片工具 | Raphael.app | 输入提示词，得到图片 |
| 本项目 | AI 练习卷工具 | 输入知识点/上传资料，得到可打印练习卷 |

### 2.4 产品原则

1. **极简**：一个页面完成核心流程。
2. **工具化**：打开即用，不做复杂账号体系。
3. **打印优先**：结果必须像一份可打印练习卷。
4. **轻后端**：上传文件和生成文件不长期保存；但商业 MVP 必须保留用户、点数、订单、分享奖励、邀请奖励和生成流水。
5. **免费可试**：用户先生成并下载带水印 PDF，再引导升级无水印、Word、答案解析和整卷仿真。
6. **成本可控**：用极简点数制控制生成、上传资料和整卷仿真成本；不让用户计算复杂费用。
7. **长期工具矩阵**：首个工具成功后，扩展为教育打印工具站。

## 3. 目标用户

### 3.1 核心用户

#### 1. 家长

典型需求：

- 孩子某类题反复错。
- 想快速生成同类题练习。
- 不会自己出题。
- 不想买整本练习册。
- 希望直接打印给孩子做。

典型场景：

> 上传孩子的一道错题或一页作业，生成 10 道同类题，带答案解析，下载 PDF 打印。

#### 2. 老师

典型需求：

- 备课时需要快速出课堂小测。
- 根据讲义生成练习题。
- 需要 Word 可编辑版本。
- 需要答案页。
- 希望节省排版时间。

典型场景：

> 上传一页讲义，生成一份课堂练习卷，下载 Word 后稍作修改。

#### 3. 家教 / 辅导老师

典型需求：

- 针对不同学生快速定制练习。
- 根据试卷/错题生成课后作业。
- 高频使用，愿意为效率付费。

典型场景：

> 根据学生今天错题，生成一份课后同类题练习卷，发给家长打印。

#### 4. 学生 / 自学用户

典型需求：

- 输入知识点，生成练习题自测。
- 上传笔记/PDF，生成自测题。
- 下载练习卷复习。

## 4. 市场判断

### 4.1 市场有需求

该需求本质是“AI 生成内容 + 标准文档输出 + 打印场景”。通用 AI 能生成题，但普通用户不会稳定完成：

```text
提示词设计 → 题目生成 → 排版 → 答案页 → PDF/Word → 打印
```

本产品把这条链路压缩为：

```text
输入/上传 → 生成 → 下载 PDF/Word
```

### 4.2 不是大平台机会，而是工具站机会

本产品不适合作为重型教育平台起步，更适合按“工具站”长期做：

- 首个工具：AI 练习卷打印助手
- 后续工具矩阵：
  - AI 口算题生成器
  - AI 英语单词默写纸
  - AI 阅读理解题生成器
  - AI 错题同类题生成器
  - AI 整卷仿真生成器
  - AI 答案解析页生成器
  - AI 试卷排版器

### 4.3 对标模式

对标的不是某个教育 SaaS，而是 Raphael.app 式工具站打法：

| Raphael.app | 本产品 |
|---|---|
| 免费生成图片 | 免费生成少量练习题 |
| 图片下载 | PDF / Word 下载 |
| 水印 | 免费 PDF 带水印 |
| Credits 点数 | 生成点数 / OCR 点数 |
| 高质量模型付费 | 整卷仿真 / Word / 无水印 / OCR 付费 |
| 工具站流量 | 教育打印工具站流量 |

核心学习点：

```text
免费带来流量
水印制造转化
点数控制成本
高级能力收费
工具矩阵扩大入口
```

5. PDF 下载

## 7. 产品形态

### 7.1 首发形态建议

优先建议：

```text
Web/H5 工具页 + 轻后端 API
```

理由：

1. 上线最快。
2. PDF / Word 下载最自然。
3. 可快速验证付费按钮。
4. 适合 SEO。
5. 后续可复用同一后端做微信小程序。

### 7.2 微信小程序形态

小程序可作为第二入口，前端只负责：

1. 输入。
2. 上传。
3. 调用后端。
4. 展示预览。
5. 打开 PDF / Word。
6. 触发付费。

小程序端不做：

- 不在小程序端调用大模型。
- 不在小程序端生成 PDF / Word。
- 不在小程序端解析文件。
- 不暴露 API Key。

### 7.3 推荐路线

```text
第 1 天：Web 极简版上线
第 2～3 天：补 PDF/Word/付费/上传
第 3～7 天：复用后端做微信小程序
```

## 9. 核心模式

适用场景：

- 用户输入知识点。
- 用户上传一页资料。
- 用户想生成 5～20 道练习题。

流程：

```text
用户输入/上传
↓
AI 提取知识点
↓
生成题目
↓
生成答案解析
↓
输出 PDF / Word
```

### 示例输入

```text
生成 10 道初一数学一元一次方程中等题，带答案解析。
```

### 示例输出

```text
初一数学一元一次方程练习卷

一、选择题
二、填空题
三、解答题

答案与解析
```

## 9.2 整卷仿真模式

适用场景：

- 用户上传一份试卷。
- 用户想得到另一份类似试卷。
- 用户希望题型结构、知识点、难度相近，但题目不同。

### 核心原则

不能复制原题。  
应生成：

```text
结构相似
知识点相似
难度相似
题型相似
内容重新生成
```

### 正确流程

```text
上传原试卷
↓
OCR / 文本解析
↓
分析原试卷结构
↓
生成试卷蓝图
↓
按蓝图逐题生成新题
↓
生成答案解析
↓
排版 PDF / Word
```

## 10. 定价策略（v1.1 最终版）

## 10.1 定价总原则

MVP 阶段不要设计复杂点数经济系统。用户只需要知道：

```text
我还剩多少点？
这次生成消耗多少点？
PDF 能不能下载？
免费版有没有水印？
付费后有什么区别？
```

最终原则：

1. **只为生成扣点**，下载 PDF 不扣点。
2. **免费 PDF 有水印**，付费版 PDF 无水印。
3. **Word 是套餐权益**，不额外扣点。
4. **整卷仿真固定 10 点**，MVP 不按页数拆分计费。
5. **国内和海外分开定价**，但点数消耗规则保持一致。
6. **点数包性价比低于会员**，用于临时加量，避免用户只买点数包不订阅。
7. **分享奖励只做两条简单规则**：每日朋友圈分享 +1；好友首次购买邀请人 +5。

---

## 10.2 点数消耗规则

用户能看到的点数规则只保留 5 个核心生成场景：

| 用户操作 | 消耗点数 |
|---|---:|
| 普通练习卷，10 题以内 | 1 点 |
| 加长练习卷，20 题以内 | 2 点 |
| 错题同类题 | 2 点 |
| 上传资料生成 | 3 点 |
| 整卷仿真 | 10 点 |

不扣点的操作：

| 用户操作 | 消耗点数 | 权益说明 |
|---|---:|---|
| 在线预览 | 0 点 | 所有用户可用 |
| 下载 PDF | 0 点 | 免费版有水印，付费版无水印 |
| 下载 Word | 0 点 | 仅 Pro / Teacher 可用 |
| 去水印 PDF | 0 点 | 仅付费版可用 |
| 查看基础答案 | 0 点 | Free 可展示基础答案 |
| 查看完整答案解析 | 0 点 | 作为 Pro / Teacher 权益，不额外扣点 |

### 展示方式

生成按钮旁边只显示一句话：

```text
本次生成将消耗 1 点
你当前剩余 12 点
```

整卷仿真时显示：

```text
本次整卷仿真将消耗 10 点
你当前剩余 12 点
```

不要让用户看到复杂扣费公式。

## 11.2 成本控制原则

MVP 不让用户计算复杂费用，但后台必须控制成本。

核心限制：

| 功能 | V1 限制 |
|---|---:|
| 普通练习卷 | 10 题以内，1 点 |
| 加长练习卷 | 20 题以内，2 点 |
| 上传资料生成 | 固定 3 点，限制页数 / 文件大小 |
| 整卷仿真 | 固定 10 点，最多 5 页 |
| PDF 下载 | 不扣点，但免费版加水印 |
| Word 下载 | 不扣点，但仅 Pro / Teacher |
| 生成失败重试 | 最多自动重试 1 次 |
| 文件存储 | 临时处理，不长期保存原文件 |

### 成本风险

最容易打穿利润的是：

```text
1. 用户上传多页扫描 PDF
2. 用户要求完整试卷仿真
3. 用户反复生成/重试
4. 用户要求 Word + PDF + 完整解析
5. 高级模型成本过高
6. 免费用户通过分享奖励长期白嫖
```

### 对策

1. 上传资料生成虽然固定 3 点，但必须限制文件大小、页数和字符数。
2. 整卷仿真固定 10 点，但最多上传 5 页。
3. 免费点数只给一次，新用户国内 3 点，海外 5 credits。
4. 每日朋友圈分享最多奖励 1 点。
5. 邀请奖励只在好友首次购买成功后发放。
6. 高成本功能优先放到 Pro / Teacher。
7. 后台记录每次生成的模式、点数、模型消耗和成功/失败状态。

7. 生成失败不扣点；扣点失败不生成。

## 20. 风险与对策

### 风险 1：大模型生成题目质量不稳定

对策：

- 强制 JSON。
- 限定题量。
- 简洁解析。
- 对答案做基础校验。
- 后续加入二次校验模型。

### 风险 2：OCR 识别不准

对策：

- MVP 上传资料生成固定 3 点，但限制页数、文件大小和清晰度。
- 优先支持 Word/PDF 文本。
- 图片 OCR 作为增强能力。
- 提示用户上传清晰图片。

### 风险 3：成本被高频用户打穿

对策：

- 只有生成扣点。
- 普通生成 1 点。
- 加长生成 2 点。
- 上传资料生成 3 点。
- 整卷仿真固定 10 点。
- 整卷最多上传 5 页。
- 每日朋友圈分享最多奖励 1 点。
- 邀请奖励只在好友首次购买成功后发放。
- 生成失败最多自动重试 1 次。

### 风险 4：用户觉得 ChatGPT/豆包也能做

对策：

强调差异：

```text
ChatGPT 给你文本。
我们给你一份能打印的练习卷。
```

### 风险 5：版权/试卷复制问题

对策：

- 不使用“复制考卷”作为正式文案。
- 使用“整卷仿真”“同结构练习卷”“同难度变式卷”。
- 不复制原题。
- 生成相似知识点和结构的新题。
- 输出免责声明。
- 不长期保存用户上传资料。
- 不提供原试卷还原服务。

### 风险 6：微信小程序审核/支付拖慢上线

对策：

- 可先做 Web/H5 验证。
- 后续复用后端做小程序。
- 支付可先用 mock 或人工收款验证。
- 小程序上线后再接微信支付和朋友圈分享奖励。

### 风险 7：点数系统过复杂导致用户理解困难

对策：

- 用户侧只展示 5 条生成消耗规则。
- PDF 下载不扣点。
- Word 不扣点，只做套餐权益。
- 不按 OCR 页数、答案页、解析页、排版拆分收费。
- 页面只显示“本次消耗 X 点，当前剩余 Y 点”。

## 21. 隐私与合规

MVP 说明：

1. 上传文件仅用于当次生成。
2. 不长期保存上传资料。
3. 不保存用户历史记录。
4. 生成结果仅供学习练习参考。
5. AI 生成内容可能存在错误，建议老师/家长检查后使用。

## 25. 最终结论

本项目应按“AI 教育打印工具站”而不是“教育平台”来做。

第一版的核心不是功能多，而是让用户感受到：

```text
我输入或上传资料后，真的得到了一份可以打印的练习卷。
```

最小商业闭环是：

```text
免费点数体验
↓
生成练习卷
↓
下载带水印 PDF
↓
升级解锁无水印 PDF / Word / 完整答案解析 / 整卷仿真
```

v1.1 最终商业规则是：

```text
只为生成扣点
PDF 下载不扣点
免费 PDF 有水印
付费 PDF 无水印
Word 是 Pro / Teacher 权益
整卷仿真固定 10 点
朋友圈每日分享 +1 点
好友首次购买邀请人 +5 点
```

国内先跑：

```text
小程序 / H5
微信支付
小红书 / 家长群 / 老师群
低价套餐 + 点数包
```

海外后续跑：

```text
Web
SEO
英文 worksheet generator 页面
美元订阅 + credits pack
```

长期目标不是一个单工具，而是教育打印工具矩阵：

```text
每个工具都很小
每个工具都能被搜索
每个工具都能带来用户
每个工具都能导向付费
```

这就是本项目最合理、最容易执行、最容易验证付费的打法。
